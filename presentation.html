Overview

Project Overview

A Clinical Decision Support System (CDSS) designed for Cardiac ICU environments. Instead of passive data storage, this system actively interprets incoming vitals to assist in early detection of life-threatening cardiac events.

  • Real-time cardiac parameter ingestion & storage
  • Automated arrhythmia detection with rule-based engines
  • HEART, TIMI & GRACE risk-scoring on every telemetry entry
  • Hemodynamic stability assessment (Stable / Unstable / Critical)
  • 24-hour time-series analysis for trend visualization
  • STEMI recognition & Heart Failure severity indexing

Why This Matters

In ICU settings, minutes decide outcomes. Early STEMI detection can reduce door-to-balloon time and significantly improve patient survival rates.

This system doesn't just store data — it interprets it, providing risk scores and alerts the moment vitals are recorded.

Data Modeling

Why MongoDB?

Medical data is inherently semi-structured. ECG rhythms, echocardiogram findings, and lab results vary between patients. MongoDB's flexible document model handles this naturally.

🔑 Key Design Decision

ScoringResult is embedded inside the CardiacTelemetry document — not in a separate collection. This means a single read returns vitals + risk scores together, avoiding expensive joins.

  • Flexible schema for varying clinical data
  • Embedded documents eliminate SQL JOINs
  • Native aggregation pipelines for analytics
  • Time-series optimized queries
  • Horizontal scaling for high-throughput ICU data

Scoring

Clinical Scoring Systems

Three gold-standard cardiac risk assessment algorithms are computed automatically on every telemetry entry.

Score Range Components Risk Stratification
HEART 0–10 History · ECG · Age · Risk Factors · Troponin — each scored 0–2 0–3 Low · 4–6 Moderate · 7+ High
TIMI 0–7 Age ≥65, ≥3 CAD factors, Known stenosis, ASA use, ≥2 angina/24h, ST deviation, Elevated troponin 0–2 Low · 3–4 Moderate · 5+ High
GRACE 0–372 Age, Heart rate, Systolic BP, Creatinine, Cardiac arrest, ST deviation, Troponin, Killip class <109 Low · 109–140 Moderate · >140 High

Combined Risk Level

If any score exceeds its "High" threshold (HEART ≥ 7, TIMI ≥ 5, GRACE ≥ 140), the overall risk level is flagged as HIGH — triggering priority alerts.

Heart Failure Severity

HF severity is independently assessed via BNP levels and Killip classification: Killip IV or BNP > 900 → Severe, Killip III or BNP > 400 → Moderate.

STEMI Recognition & Arrhythmia Detection

🚨 STEMI Recognition — Critical

ST-Elevation Myocardial Infarction is the #1 time-critical cardiac emergency. The system auto-detects STEMI by pattern-matching the ECG rhythm field.

Time-Series Analysis & Pattern Recognition

24h Time-Series Pipeline

# MongoDB Aggregation — database.py pipeline = [ {"$match": { "patient_id": patient_id, "timestamp": {"$gte": cutoff_24h} }}, {"$sort": {"timestamp": 1}}, {"$project": { "_id": 0, "timestamp": 1, "heart_rate": 1, "systolic_bp": 1, "ecg_rhythm": 1, "troponin_level": 1 }} ]

This pipeline feeds the Streamlit line charts — allowing ICU staff to see if a patient's heart rate and blood pressure are stabilizing or deteriorating over time.

Arrhythmia Pattern Recognition

# Pattern aggregation — database.py pipeline = [ {"$match": { "severity": { "$in": ["Medium","High","Critical"] } }}, {"$group": { "_id": { "type": "$type", "severity": "$severity" }, "count": {"$sum": 1}, "latest": {"$max": "$timestamp"}, "rules": {"$addToSet": "$detection_rule"} }}, {"$sort": {"count": -1}} ]

Groups arrhythmia alerts by type and severity, counts occurrences, and identifies recurring patterns — valuable for long-term treatment planning.
